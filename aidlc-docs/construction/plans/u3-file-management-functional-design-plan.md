# U3: File Management — Functional Design プラン

## ユニット情報
- **ユニット**: U3 File Management
- **MVPストーリー**: US-09（ワークスペース管理）, US-10（ファイル操作）, US-11（複数タブ編集）
- **依存ユニット**: U1 Foundation（完了済み）, U2 Core Editor（完了済み）
- **含むコンポーネント**: C2 FileManager, C8 UIShell（サイドバー、タブバー）, S2 WorkspaceService
- **既存基盤**: FileSystemAdapter（U1で抽象定義済み）, SettingsManager, DirEntry/FileContent型

---

## 機能設計ステップ

### Step 1: ドメインエンティティ設計
- [x] Workspace エンティティ（パス、名前、ファイルツリー、最終アクセス日時）
- [x] Tab エンティティ（ファイルパス、表示名、未保存フラグ、アクティブフラグ、スクロール位置）
- [x] FileTreeNode エンティティ（DirEntry拡張: 展開/折り畳み状態、ソート順）
- [x] RecentWorkspace エンティティ（パス、名前、最終アクセス日時）
- [x] SessionState エンティティ（ワークスペースパス、開いているタブ、アクティブタブ、カーソル位置）

### Step 2: ビジネスロジック設計 — FileManager
- [x] openWorkspace(path): フォルダ読み込み → ファイルツリー構築 → 状態更新
- [x] closeWorkspace(): 未保存チェック → セッション保存 → 状態クリア
- [x] buildFileTree(path): 再帰ディレクトリ読み込み → DirEntry→FileTreeNode変換 → ソート
- [x] createFile(parentPath, name): バリデーション → ファイル作成 → ツリー更新
- [x] saveFile(path, content): 書き込み → 保存ステータス更新 → 未保存フラグ解除
- [x] renameFile(oldPath, newPath): バリデーション → リネーム → タブ/ツリー更新
- [x] deleteFile(path): 確認 → 削除 → タブ閉じ → ツリー更新
- [x] createFolder(parentPath, name): バリデーション → フォルダ作成 → ツリー更新

### Step 3: ビジネスロジック設計 — タブ管理
- [x] openTab(path): ファイル読み込み → タブ追加/アクティブ化 → EditorCoreに内容設定
- [x] closeTab(path): 未保存チェック → タブ除去 → 隣接タブをアクティブ化
- [x] switchTab(path): コンテンツ保存 → アクティブタブ切替 → EditorCoreに内容設定
- [x] markDirty(path): 未保存フラグ設定
- [x] getOpenTabs(): タブ一覧取得
- [x] reorderTabs(from, to): タブ順序変更（D&D対応判断含む）

### Step 4: ビジネスロジック設計 — WorkspaceService
- [x] initialize(): 前回セッション復元 → ワークスペース開く → タブ復元
- [x] saveSession(): 現在の状態をSettingsManagerに永続化
- [x] restoreSession(): SettingsManagerから状態読み込み → FileManagerで復元
- [x] requestClose(): 全タブ未保存チェック → 確認ダイアログ → 結果返却

### Step 5: ビジネスルール定義
- [x] ファイル名バリデーションルール（禁止文字、最大長、重複チェック）
- [x] ワークスペース深さ制限（ファイルツリーの最大再帰深度）
- [x] タブ最大数制限（同時に開けるタブの上限）
- [x] 未保存警告ルール（タブ閉じ/ワークスペース閉じ/アプリ終了時）
- [x] ファイルフィルタリングルール（表示するファイル種別、隠しファイル扱い）
- [x] ファイルソートルール（フォルダ優先、アルファベット順等）

### Step 6: フロントエンドコンポーネント設計
- [x] FileTree.svelte — ファイルツリー表示（展開/折り畳み、コンテキストメニュー）
- [x] FileTreeItem.svelte — 個別ファイル/フォルダアイテム（アイコン、選択状態）
- [x] TabBar.svelte — タブバー（タブ一覧、アクティブ表示、閉じるボタン）
- [x] Tab.svelte — 個別タブ（ファイル名、未保存インジケーター、閉じるボタン）
- [x] NewFileDialog.svelte — 新規ファイル/フォルダ作成ダイアログ
- [x] RenameDialog.svelte — リネームダイアログ

### Step 7: ストア設計
- [x] workspace.svelte.ts — ワークスペース状態ストア（$state）
- [x] tabs.svelte.ts — タブ管理ストア（$state）
- [x] file-tree.svelte.ts — ファイルツリー状態ストア（$state）

## 質問

以下の質問にお答えください。各質問の `[Answer]:` の後に回答を記入してください。

### Q1: ファイルツリーの表示範囲
ファイルツリーに表示するファイルの範囲はどうしますか？

A) Markdownファイル（.md）のみ表示
B) 全ファイルを表示（隠しファイルは除く）
C) 全ファイルを表示（隠しファイル含む）
D) フィルタリング機能で切り替え可能にする（デフォルトは全ファイル表示、隠しファイル除外）

[Answer]:D

### Q2: ファイルツリーの再帰深度
ファイルツリーの最大深度（ネスト階層数）はどの程度を想定していますか？

A) 3階層まで（ワークスペース直下+2階層）
B) 5階層まで
C) 10階層まで
D) 制限なし（パフォーマンスに応じて遅延読み込み）

[Answer]:D

### Q3: タブの最大数
同時に開けるタブの上限はありますか？

A) 制限なし
B) 10タブまで
C) 20タブまで
D) 制限は設けるが設定で変更可能

[Answer]:D

### Q4: ファイルツリーでのコンテキストメニュー
ファイルツリー上で右クリックしたときの操作メニューはどこまで実装しますか？

A) 基本操作のみ（新規作成、名前変更、削除）
B) A + コピー/移動
C) A + パスのコピー
D) A + B + C（フル機能）

[Answer]:D

### Q5: 未保存変更の自動保存
EditorCore（U2）にはデバウンス自動保存（1000ms）が実装済みですが、U3のファイル保存との連携はどうしますか？

A) EditorCoreの自動保存コールバックでFileManager.saveFile()を呼び出し、ファイルに書き込む（完全自動保存）
B) EditorCoreの自動保存はメモリ上の状態保持のみ。ファイルへの書き込みはCtrl+S の明示的操作のみ
C) 両方対応（設定で切り替え可能）

[Answer]:C

### Q6: タブ切替時のエディター状態保持
タブを切り替えた際、前のタブのスクロール位置やカーソル位置を保持しますか？

A) はい、カーソル位置とスクロール位置の両方を保持
B) カーソル位置のみ保持
C) 保持しない（タブ切替時は常にファイル先頭に戻る）

[Answer]:A

### Q7: タブのドラッグ&ドロップ並び替え
タブの並び替えをドラッグ&ドロップで行えるようにしますか？

A) MVP で実装する
B) Post-MVP で実装する（MVP ではタブ順は開いた順に固定）

[Answer]:A

### Q8: ワークスペース外ファイルの扱い
ワークスペース外のファイルを開けるようにしますか（例: ドラッグ&ドロップでファイルを持ち込む）？

A) ワークスペース内のファイルのみ開ける
B) ワークスペース外のファイルも開けるが、保存はワークスペース内にコピー
C) ワークスペース外のファイルもその場で開いて編集可能

[Answer]:B

### Q9: セッション復元の範囲
アプリ再起動時のセッション復元で何を復元しますか？

A) ワークスペースパス + 開いていたタブ一覧のみ
B) A + アクティブだったタブ
C) A + B + 各タブのカーソル位置/スクロール位置
D) A + B + C + サイドバーの展開/折り畳み状態

[Answer]:C

### Q10: 新規ファイルの作成方法
新規ファイルの作成UIはどの方式を採用しますか？

A) サイドバー上部に「＋」ボタン → ファイル名入力ダイアログ
B) ファイルツリー上のコンテキストメニューから「新規作成」→ インライン編集（ツリー内で直接ファイル名入力）
C) 両方対応

[Answer]:C

### Q11: 削除操作の対象
削除操作はフォルダも対象にしますか？

A) ファイルのみ削除可能（フォルダ削除はMVPでは対象外）
B) ファイルとフォルダの両方を削除可能（フォルダは中身ごと削除、確認ダイアログ付き）

[Answer]:B

### Q12: Markdownファイル以外の編集
Markdown以外のファイル（.txt, .json等）をエディターで開いた場合の動作は？

A) Markdownファイル（.md, .mdx）のみ開ける。それ以外はクリックしても無反応
B) テキストファイルは全てプレーンテキストとして開ける（WYSIWYG変換なし）
C) Markdownファイルのみ開ける。それ以外はクリック時にエラー通知表示

[Answer]:B
