## 質問

以下の質問にお答えください。

### Q1: ファイルツリーの規模想定
ワークスペース内のファイル/フォルダ数はどの程度を想定していますか？（パフォーマンス基準の設定に影響）

A) 小規模（〜100ファイル程度、個人ブログ）
B) 中規模（〜1,000ファイル程度、複数プロジェクト）
C) 大規模（〜10,000ファイル程度、企業レベル）
D) B+Cの両方を考慮（中規模をメインターゲット、大規模でも動作保証）

[Answer]:A

### Q2: ファイル監視（外部変更検出）
ワークスペース内のファイルが外部エディター等で変更された場合の検出は必要ですか？

A) MVPでは不要（アプリ内の操作のみ想定）
B) ファイル変更を検出し、再読み込みを促す通知を表示
C) 自動的に最新の内容に更新（衝突時は確認ダイアログ）

[Answer]:C

### Q3: クラッシュリカバリ
アプリが予期せずクラッシュした場合の未保存データ保護はどの程度必要ですか？

A) セッション情報のみ保持（開いていたタブ一覧の復元）
B) A + 未保存コンテンツのリカバリファイル保存（定期的にtmpに書き出し）
C) MVPではA のみ、Post-MVPでBを追加

[Answer]:B

### Q4: 大規模フォルダの扱い
1つのフォルダ内に大量のファイル（例: 1,000+）がある場合の対応方針は？

A) 全ファイルを表示（パフォーマンス低下は許容）
B) 一定数（例: 200件）で切り捨て + 「さらに表示」ボタン
C) 仮想スクロール実装でDOM数を制限

[Answer]:C

### Q5: ファイルサイズ制限
エディターで開けるファイルのサイズ上限はどうしますか？

A) U2のドキュメント制限に準拠（200KB推奨、1MB最大）
B) プレーンテキストはより大きいサイズも許容（例: 5MB）
C) Aのみ（プレーンテキストもMarkdownと同じ制限）

[Answer]:A

### Q6: パストラバーサル防止の厳密性
ワークスペース外ファイルへのアクセス制限はどの程度厳密にしますか？

A) FileManagerレベルでワークスペースルートからの相対パスを検証（`..`を含むパスを拒否）
B) A + FileSystemAdapterレベルでもダブルチェック
C) Tauriのscope機能で制限（アプリレベルでワークスペースフォルダのみアクセス許可）
D) B + C の多層防御

[Answer]:D

### Q7: アクセシビリティレベル
U2ではWCAG 2.1 AAAを目標としましたが、U3のファイルツリー/タブバーも同レベルを目指しますか？

A) AAA（U2と同レベル）
B) AA（実用的なレベル）

[Answer]:A
