# U1: Foundation - ビジネスルール

## FileSystemAdapter ルール

### BR-FS-01: エラーハンドリング
- ファイルが存在しない場合: `FileSystemError { code: 'NOT_FOUND' }` を返す
- 権限がない場合: `FileSystemError { code: 'PERMISSION_DENIED' }` を返す
- ファイルが既に存在する場合（作成時）: `FileSystemError { code: 'ALREADY_EXISTS' }` を返す
- Web版でサポートされない操作: `FileSystemError { code: 'NOT_SUPPORTED' }` を返す
- その他のI/Oエラー: `FileSystemError { code: 'IO_ERROR' }` を返す

### BR-FS-02: パス正規化
- パス区切り文字はプラットフォームに依存せず `/` を使用（Tauri側で変換）
- 相対パスは許可しない（常に絶対パスを使用）

### BR-FS-03: ファイルサイズ制限
- 読み込みファイルサイズの上限: 50MB（テキスト）、100MB（バイナリ/画像）
- 上限を超える場合はエラーを返す

### BR-FS-04: ディレクトリ読み込み
- readDir は1階層のみ読み込む（再帰しない）
- UIツリー展開時に子ディレクトリをオンデマンドで読み込む（パフォーマンス）
- 隠しファイル（`.`で始まるファイル）はデフォルトで非表示

## SettingsManager ルール

### BR-SET-01: 設定バリデーション
- `fontSize`: 8 以上 48 以下の整数。範囲外はデフォルト値(16)にリセット
- `theme`: `'light'` または `'dark'` のみ。不正値はデフォルト値('light')にリセット
- `editorWidth`: 400 以上 1200 以下の整数。範囲外はデフォルト値(720)にリセット
- `recentWorkspaces`: 最大10件。超過時は古いものから削除

### BR-SET-02: 設定ファイル破損時のリカバリ
- JSONパースに失敗した場合: デフォルト設定で上書き（データ消失を通知）
- スキーマ不一致の場合: デフォルト値でマージ（既存の有効なキーは保持）

### BR-SET-03: 設定保存のデバウンス
- 設定変更後、500ms間追加の変更がなければファイルに書き込む
- 複数の設定を連続変更した場合、最後の状態のみ書き込む
- アプリ終了時は即時書き込み（デバウンスをフラッシュ）

### BR-SET-04: 最近のワークスペース管理
- ワークスペースを開くたびにリストの先頭に移動
- 既に存在する場合は重複せず先頭に移動
- 最大10件を超えたら末尾を削除
- 存在しないパスはリストから自動除去（次回アクセス時）

## SecureStorage ルール

### BR-SEC-01: 認証情報の保存
- 保存キー形式: `"platform:{platformId}"` （例: `"platform:zenn"`）
- 値は `PlatformCredentials` をJSON文字列化して保存
- Tauri版: tauri-plugin-stronghold で暗号化保存
- Web版: localStorage にbase64エンコードで保存（暗号化なし）

### BR-SEC-02: Web版セキュリティ警告
- Web版で初めてAPIキーを保存する際に警告を1回表示
- 警告内容: 「Web版ではAPIキーがブラウザのローカルストレージに保存されます。共有端末での利用にはご注意ください。」
- ユーザーが「了解」をクリックした後に保存を実行
- 警告表示済みフラグを設定に保存（再表示しない）

## UIShell ルール

### BR-UI-01: サイドバー
- デフォルト状態: 非表示（Typora風、執筆集中モード）
- トグルショートカット: `Ctrl/Cmd + \`
- サイドバー幅: 260px（固定、将来的にリサイズ対応可）
- アニメーション: 200ms ease-in-out

### BR-UI-02: エディターエリア
- 最大幅は `editorWidth` 設定値に従う（デフォルト720px）
- 常に水平中央配置
- 上下パディング: 2rem
- サイドバー表示時もエディターは中央配置を維持

### BR-UI-03: ツールバー
- アプリ上部に固定表示
- 左側: サイドバートグルボタン、ファイル名表示
- 中央: 書式ボタン（太字、斜体、見出し等）— EditorCoreが提供するコマンドに連動
- 右側: 投稿ボタン、設定ボタン

### BR-UI-04: ステータスバー
- アプリ下部に固定表示
- 表示項目: 文字数、行数、保存状態（保存済み / 未保存）、接続状態
- 高さ: 24px

### BR-UI-05: 通知
- 成功/エラー/情報/警告の4種類
- 画面右下にトースト表示
- デフォルト表示時間: success=3000ms, info=5000ms, warning=5000ms, error=0(手動閉じ)
- 同時表示は最大3件、超過分はキューに入れる
