# U2: Core Editor — ビジネスルール

## BR-01: 書式適用ルール

### BR-01.1: インライン書式（Mark）
- テキスト選択中に書式コマンドを実行すると、選択範囲にMarkを適用/解除（トグル）
- テキスト未選択でコマンドを実行すると、次に入力するテキストにMarkが適用される（storedMark）
- 同じインライン書式を重複適用しない（既に太字の範囲に太字コマンド → 解除）

### BR-01.2: ブロック書式（Node）
- カーソル位置のブロック全体にNode変換を適用
- 見出しレベル変更: 同レベル見出しコマンド → 段落に戻す（トグル動作）
- リスト ↔ 段落: リストコマンドをリスト内で実行 → リスト解除

### BR-01.3: ネスト可能なブロック
- リスト: Tab でネスト深化（最大6レベル）、Shift+Tab で浅化
- 引用ブロック: ネスト可能（引用内引用）
- リスト内Enter: 新しいリストアイテム追加
- リスト内空行Enter: リスト解除して段落に戻る

## BR-02: Markdown自動変換ルール（InputRule）

### BR-02.1: ブロックレベルInputRule
| 入力パターン | 変換先 | 条件 |
|---|---|---|
| `# ` | H1 | 行頭のみ |
| `## ` | H2 | 行頭のみ |
| `### ` | H3 | 行頭のみ |
| `#### ` | H4 | 行頭のみ |
| `##### ` | H5 | 行頭のみ |
| `###### ` | H6 | 行頭のみ |
| `- ` / `* ` | 箇条書きリスト | 行頭のみ |
| `1. ` | 番号付きリスト | 行頭のみ |
| `- [ ] ` / `- [x] ` | タスクリスト | 行頭のみ |
| `> ` | 引用ブロック | 行頭のみ |
| ` ``` ` | コードブロック | 行頭のみ |
| `---` / `***` | 水平線 | 行頭、3文字以上 |

### BR-02.2: インラインInputRule
| 入力パターン | 変換先 | 条件 |
|---|---|---|
| `**text**` | 太字 | 閉じ記号入力時 |
| `*text*` / `_text_` | 斜体 | 閉じ記号入力時 |
| `~~text~~` | 取り消し線 | 閉じ記号入力時 |
| `` `code` `` | インラインコード | 閉じバッククォート入力時 |

### BR-02.3: 自動リンク化（PasteRule）
- URLパターン（`https://...`）のペーストでリンクMarkを自動適用
- テキスト選択中にURLをペースト → 選択テキストをリンクテキストとしてリンク作成
- テキスト未選択でURLペースト → URL自体がリンクテキスト

## BR-03: スラッシュコマンドルール

### BR-03.1: トリガー条件
- `/` 入力でコマンドパレット表示
- 条件: 行頭、または空白文字の直後
- コードブロック内では無効（プレーンテキストとして入力）

### BR-03.2: フィルタリングロジック
- クエリ文字列で前方一致フィルタ
- エイリアスも検索対象（例: "h1" → "見出し1"、"list" → "箇条書きリスト"）
- 大文字小文字を区別しない
- マッチ0件 → パレット非表示（`/` テキストはそのまま残る）

### BR-03.3: 操作ルール
- 矢印キー ↑↓: 選択移動（循環あり）
- Enter: 選択中のコマンド実行、`/` + クエリ文字列を削除してノード挿入
- Escape: パレット閉じる、`/` テキストは残す
- クリック: コマンド実行（Enterと同じ動作）

## BR-04: テーブル操作ルール

### BR-04.1: テーブル挿入
- デフォルトサイズ: 3行 x 3列（ヘッダー行含む）
- ヘッダー行は太字＋背景色（CSSスタイル）

### BR-04.2: セル移動
- Tab: 次のセルへ（行末→次行先頭、テーブル末尾→新しい行を追加）
- Shift+Tab: 前のセルへ
- 矢印キー: セル境界でのみ隣接セルへ移動

### BR-04.3: 行列操作
- 行/列の追加: 現在のセル位置の後ろに追加
- 行/列の削除: 現在のセルが含まれる行/列を削除
- テーブル削除: テーブル全体を削除して段落に戻す
- 最後の行/列の削除 → テーブル全体削除

### BR-04.4: セル揃え
- テーブルヘッダーまたはツールバーから left/center/right を切り替え
- デフォルト: left

## BR-05: コードブロックルール

### BR-05.1: 言語設定
- コードブロック挿入時のデフォルト言語: `plaintext`
- コードブロック上部にドロップダウンで言語変更可能
- ```` ```lang ```` 入力で言語指定付きコードブロック挿入

### BR-05.2: コードブロック内の操作
- コードブロック内ではエディターショートカット（Ctrl+B等）を無効化
- Tab: インデント（スペース2文字）
- Shift+Tab: アンインデント
- Enter: 改行（リストアイテム追加ではない）

### BR-05.3: コードブロックからの脱出
- 最終行で下矢印キー → コードブロック後の段落にカーソル移動
- Escape → コードブロック後の段落にカーソル移動
- コードブロック後に段落がない場合 → 自動で空段落を作成

## BR-06: リンク編集ルール

### BR-06.1: リンク作成
- テキスト選択 → ツールバーまたはバブルメニューのリンクボタン → インラインポップオーバー表示
- URL入力 + Enter で確定
- 空URL → リンク解除

### BR-06.2: リンク編集
- 既存リンクをクリック → ポップオーバー表示（URL表示 + 編集/削除ボタン）
- 編集ボタン → URL編集モード
- 削除ボタン → リンクMark解除（テキストは残す）

### BR-06.3: 自動リンク化
- URLパターンの入力完了（スペースまたはEnter）で自動的にリンクMarkを適用
- 自動リンク化されたリンクのテキスト編集 → URLも連動更新

## BR-07: Undo/Redo ルール

### BR-07.1: 履歴管理
- Tiptap History拡張（ProseMirror history プラグイン）がデフォルトで管理
- 一定時間内の連続入力は1つの履歴ステップにまとめる（デフォルト500ms）
- 書式変更は独立した履歴ステップ

### BR-07.2: 履歴のライフサイクル
- ファイル保存しても履歴は保持される（セッション中）
- ファイル切り替え（タブ切り替え）: 各タブが独立した履歴を持つ
- アプリ再起動: 履歴はクリア

### BR-07.3: Undo不可能な操作
- ファイル保存操作自体はUndo対象外（副作用）
- 設定変更はUndo対象外

## BR-08: PluginSystem ルール

### BR-08.1: プラグイン読み込み
- 同梱プラグインのみサポート（MVP）
- プラグインは遅延ロード（動的 import）
- ロード失敗時: エラー通知を表示、エディターは正常動作を継続

### BR-08.2: 有効/無効管理
- SettingsManager に有効/無効状態を永続化
- プラグイン無効化 → エディターからエクステンション除去（要エディター再初期化）
- デフォルト: 全同梱プラグイン有効

### BR-08.3: プラグイン間の依存
- MVP時点ではプラグイン間の依存関係なし
- 各プラグインは独立して動作
