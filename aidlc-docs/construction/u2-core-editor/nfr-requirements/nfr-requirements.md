# U2: Core Editor — NFR Requirements

## パフォーマンス要件

### NFR-U2-01: 入力遅延（Input Latency）
- **目標**: キーストロークからレンダリング完了まで50ms以下
- **測定方法**: `performance.now()` でキーダウンイベントからDOM更新完了までを計測
- **条件**: 200KB（〜20,000文字）以下のドキュメント
- **劣化許容**: 200KB超のドキュメントでは100ms以下を許容

### NFR-U2-02: エディター初期化時間
- **目標**: Tiptapエディターインスタンス生成〜初回レンダリングまで200ms以下
- **配分**:
  - Tiptap Editor + StarterKit初期化: < 100ms
  - 追加エクステンション登録（Table, CodeBlockLowlight, Link等）: < 50ms
  - 初期コンテンツのパース（空ドキュメント/プレースホルダー）: < 50ms
- **注**: U1起動バジェット残り540msのうち200msをU2に配分

### NFR-U2-03: ドキュメントサイズ上限
- **安定動作保証**: 200KB / 20,000文字
- **最大許容**: 1MB / 100,000文字（性能劣化を許容）
- **超過時**: 警告通知を表示（「ドキュメントが大きいため、動作が遅くなる可能性があります」）

### NFR-U2-04: スラッシュコマンドパレット応答
- **表示まで**: `/` 入力から50ms以内にパレット表示
- **フィルタリング**: テキスト入力から16ms以内にリスト更新（60fps）
- **コマンド実行**: 選択からノード挿入まで50ms以内

### NFR-U2-05: シンタックスハイライト性能
- **ハイライト処理**: コードブロック500行以下 → 100ms以内
- **言語切替**: ドロップダウンから言語変更後 → 100ms以内に再ハイライト
- **バンドルサイズ**: lowlight + 16言語で100KB以下（gzip後）

### NFR-U2-06: Markdown変換性能
- **ProseMirrorDoc → Markdown**: 200KB文書で100ms以内（保存時）
- **Markdown → ProseMirrorDoc**: 200KB文書で200ms以内（読み込み時）

## 信頼性要件

### NFR-U2-07: エクステンションエラー隔離
- エクステンション初期化失敗: エラーを捕捉し、該当エクステンションなしでエディター起動
- エラー通知: 「[プラグイン名]の読み込みに失敗しました」をNotification表示
- エディター本体は影響を受けず正常動作を継続

### NFR-U2-08: ドキュメントデータ保全
- ProseMirrorドキュメント構造が壊れた場合: 最後の正常な状態にロールバック
- Markdown変換で不正な出力が検出された場合: 元のProseMirrorDocを保持、エラー通知
- ブラウザクラッシュ対策: 自動保存（デバウンス1秒）で直近の状態をファイル保存

### NFR-U2-09: プラグインロードエラーハンドリング
- 遅延ロード失敗（ネットワークエラー等）: 3回までリトライ（指数バックオフ）
- 最終的に失敗: プラグイン無効化、エラー通知表示
- プラグインランタイムエラー: try-catchで捕捉、該当プラグインの該当機能のみ無効化

## ユーザビリティ要件

### NFR-U2-10: アクセシビリティ（WCAG 2.1 AAA準拠）
- **エディター領域**: `role="textbox"`, `aria-multiline="true"`, `aria-label="エディター"`
- **ツールバー**: `role="toolbar"`, 各ボタンに `aria-label` + `aria-pressed`（アクティブ状態）
- **バブルメニュー**: `role="toolbar"`, テキスト選択時のみ表示、フォーカストラップ
- **スラッシュコマンド**: `role="listbox"`, 各アイテム `role="option"`, `aria-selected`
- **コードブロック言語選択**: `role="combobox"`, 選択肢は `role="option"`
- **リンクポップオーバー**: `role="dialog"`, `aria-label="リンク編集"`, Escapeで閉じる
- **テーブル操作**: `role="menu"`, 各操作 `role="menuitem"`
- **フォーカス管理**: 全UI要素がキーボード操作可能、フォーカス順序が論理的
- **コントラスト比**: テキスト/背景のコントラスト比 7:1 以上（AAA基準）
- **フォーカスインジケーター**: 全フォーカス可能要素に視認可能なフォーカスリング
- **スクリーンリーダー対応**: 書式変更をライブリージョン（`aria-live="polite"`）で通知

### NFR-U2-11: キーボードナビゲーション
- ツールバー: Tab でツールバーにフォーカス、矢印キーでボタン間移動、Enter/Spaceで実行
- スラッシュコマンド: ↑↓ で移動、Enter で選択、Escape でキャンセル
- テーブル: Tab/Shift+Tab でセル間移動
- コードブロック: 下矢印/Escape で脱出
- ダイアログ/ポップオーバー: Escape で閉じる、Tab でフォーカストラップ内移動

### NFR-U2-12: レスポンシブ対応
- エディター幅: SettingsManager の `editorWidth` 設定に従う（400-1200px）
- ツールバー: 幅不足時はオーバーフローメニュー（「...」ボタン）に収納
- サイドバー非表示時: エディター幅が100%に拡大

## 保守性要件

### NFR-U2-13: コード品質（U1-07を継承）
- TypeScript strict mode有効
- 各エクステンション/コンポーネントにユニットテスト（カバレッジ目標: 80%以上）
- エディターコアのインテグレーションテスト（書式適用→Markdown出力の一致確認）

### NFR-U2-14: エクステンション拡張性
- 全Tiptapエクステンションは独立ファイルで実装（1ファイル1エクステンション）
- PluginSystemを介したエクステンション追加が既存コードに影響しない
- エクステンション間の依存関係は型定義で明示

---

## セキュリティ要件

### NFR-U2-15: XSSプロテクション
- ペースト時のHTMLサニタイズ: Tiptapのデフォルトサニタイザーに依存（許可されたノード/マークのみ）
- ユーザー入力のリンクURL: `javascript:` プロトコルを拒否
- コードブロック内容: HTMLエスケープ済みで表示（highlight.jsが処理）
