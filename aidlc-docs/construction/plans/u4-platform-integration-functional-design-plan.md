# U4 Platform Integration — Functional Design プラン

## 概要
U4は、プラットフォーム連携（Zenn投稿）、画像管理、エクスポート機能、投稿オーケストレーション、接続設定UIの実装を担当する。

## 対象コンポーネント
| ID | コンポーネント | 責務 |
|---|---|---|
| C4 | PlatformAdapter (ZennAdapter) | Zenn連携の実装 |
| C5 | ImageManager | 画像挿入・アップロード |
| C6 | ExportService | Markdown/HTML変換、クリップボードコピー |
| S1 | PublishService | 投稿オーケストレーション |
| C7 | SettingsManager（認証情報管理拡張） | プラットフォーム認証設定UI |
| C8 | UIShell（投稿ダイアログ、設定画面） | 投稿UI、接続設定UI |

## 対象ストーリー（MVP）
| ID | ストーリー |
|---|---|
| US-12 | Zenn記事投稿 |
| US-16 | プラットフォーム向けエクスポート |
| US-17 | 画像挿入 |
| US-18 | プラットフォーム画像アップロード |
| US-22 | プラットフォーム接続設定 |

## 設計ステップ

### Step 1: ドメインエンティティ設計
- [x] PlatformConnection（プラットフォーム接続情報）エンティティ定義
- [x] Article（記事ペイロード）エンティティ定義
- [x] ImageAsset（画像アセット）エンティティ定義
- [x] PublishRecord（投稿履歴）エンティティ定義
- [x] ExportFormat（エクスポート形式）エンティティ定義
- [x] エンティティ間のリレーションシップ図

### Step 2: PlatformAdapter / ZennAdapter ビジネスロジック設計
- [x] PlatformAdapterインターフェースの詳細設計（認証、投稿、画像アップロード）
- [x] ZennAdapter固有のビジネスロジック（GitHubリポジトリ連携方式 vs API方式の決定）
- [x] 認証フロー設計（トークン管理、接続テスト）
- [x] 記事投稿フロー設計（下書き保存、更新、公開設定）
- [x] エラーハンドリングとリトライ戦略

### Step 3: ImageManager ビジネスロジック設計
- [x] 画像受け取りフロー設計（D&D、ペースト、ファイル選択）
- [x] ローカル保存ロジック（ワークスペース内画像フォルダ管理）
- [x] 画像パス解決ロジック（相対パス/絶対パス）
- [x] プラットフォームアップロードフロー設計（バッチアップロード、進捗管理）
- [x] アップロード後のURL置換ロジック

### Step 4: ExportService ビジネスロジック設計
- [x] ProseMirror Doc → Markdown変換ロジック
- [x] ProseMirror Doc → HTML変換ロジック
- [x] プラットフォーム固有フォーマット変換（Zenn Markdown独自記法対応）
- [x] クリップボードコピーフロー
- [x] ファイルダウンロードフロー

### Step 5: PublishService オーケストレーション設計
- [x] 投稿フロー全体の状態管理（ステップバイステップ）
- [x] エクスポート→画像アップロード→記事投稿のパイプライン設計
- [x] 部分失敗時のロールバック/リカバリ戦略
- [x] 進捗通知の設計

### Step 6: ビジネスルール定義
- [x] 投稿バリデーションルール（タイトル必須、本文空チェック等）
- [x] 画像バリデーションルール（サイズ制限、形式制限）
- [x] 認証バリデーションルール（トークン有効期限チェック等）
- [x] エクスポートバリデーションルール
- [x] プラットフォーム固有の制約（Zenn文字数制限等）

### Step 7: フロントエンドコンポーネント設計
- [x] 投稿ダイアログコンポーネント設計（タイトル、タグ、公開設定）
- [x] 接続設定画面コンポーネント設計（プラットフォーム一覧、認証情報入力）
- [x] 画像挿入UI設計（D&Dオーバーレイ、挿入プレビュー）
- [x] エクスポートメニュー設計（コピー、ダウンロード）
- [x] 進捗表示・通知コンポーネント設計
- [x] ストア設計（publishStore、platformStore、imageStore、notificationStore）
