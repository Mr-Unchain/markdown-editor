# U3: File Management — ビジネスルール

## BR-U3-01: ファイル名バリデーション

### 禁止文字
- Windows禁止文字: `\ / : * ? " < > |`
- 制御文字: ASCII 0-31
- 先頭・末尾のスペース
- 先頭・末尾のドット

### 最大長
- ファイル名: 255文字（拡張子含む）
- パス全体: 260文字（Windows制限対応）

### 予約名（Windows）
- `CON`, `PRN`, `AUX`, `NUL`, `COM1`-`COM9`, `LPT1`-`LPT9`
- これらの名前（拡張子の有無問わず）は使用不可

### 重複チェック
- 同一ディレクトリ内に同名のファイル/フォルダが存在する場合はエラー
- 大文字小文字の区別はOSに依存（Windows: 区別なし）

### バリデーション結果
```typescript
interface ValidationResult {
  isValid: boolean
  error?: 'EMPTY' | 'TOO_LONG' | 'INVALID_CHARS' | 'RESERVED_NAME' | 'ALREADY_EXISTS' | 'PATH_TOO_LONG'
  message?: string
}
```

## BR-U3-02: ファイルフィルタリング

### デフォルトフィルタ
- 隠しファイル（`.`始まり）: 非表示
- 拡張子フィルタ: なし（全ファイル表示）

### フィルタ切り替え
- サイドバーヘッダーにフィルタトグルボタンを配置
- 隠しファイル表示/非表示の切り替え
- 拡張子フィルタの設定（将来拡張用にインターフェースのみ定義）

### フィルタ適用ルール
- フォルダはフィルタに関係なく常に表示（拡張子フィルタの対象外）
- 隠しフォルダ（`.git`等）は`showHiddenFiles`設定に従う

---

## BR-U3-03: ファイルソート

### ソート順（固定）
1. ディレクトリ → ファイル（ディレクトリが常に先）
2. 同種内ではロケール対応アルファベット順（`String.prototype.localeCompare`使用）
3. 大文字小文字を区別しない

## BR-U3-04: タブ制御

### 最大タブ数
- デフォルト: 20
- 設定で変更可能（最小: 1, 最大: 50）
- 上限到達時の動作:
  1. `isDirty=false` のタブのうち最も古いもの（開いた順）を自動クローズ
  2. 全タブが `isDirty=true` の場合: 通知「タブの上限に達しました。保存してから新しいタブを開いてください」

### タブ閉じ時の隣接選択ルール
1. 閉じたタブの右隣のタブを選択
2. 右隣がない場合は左隣を選択
3. 最後のタブを閉じた場合はタブなし状態（ウェルカム画面表示）

### 同一ファイルの重複オープン禁止
- 既に開いているファイルをオープンした場合、既存タブをアクティブにする
- 新しいタブは作成しない

## BR-U3-05: 未保存変更の警告

### 警告が発生するタイミング
1. **タブ閉じ**: isDirty=true のタブを閉じる際
2. **ワークスペース閉じ**: isDirty=true のタブが1つ以上ある際
3. **アプリ終了**: isDirty=true のタブが1つ以上ある際
4. **ワークスペース切替**: 別のワークスペースを開く際にisDirty=trueのタブがある場合

### 警告ダイアログの選択肢

#### タブ閉じ時
- 「保存」→ ファイル保存 → タブ閉じ
- 「保存しない」→ タブ閉じ（変更破棄）
- 「キャンセル」→ タブを閉じない

#### ワークスペース閉じ/アプリ終了時
- 「すべて保存して閉じる」→ 全未保存ファイル保存 → 閉じる
- 「保存せずに閉じる」→ 閉じる（変更破棄）
- 「キャンセル」→ 閉じない

### 警告の例外
- ファイル削除確認済みのタブ: 未保存警告なしでクローズ
- 自動保存モード（auto）: isDirtyが常にfalseのため警告不要

## BR-U3-06: ワークスペース外ファイル

### インポート方式
- ワークスペース外のファイルを開いた場合、`isExternal=true`としてタブに表示
- 保存操作時にワークスペースルート直下にコピー

### 同名ファイルの連番付与
- コピー先に同名ファイルが存在する場合:
  - `filename.md` → `filename(1).md` → `filename(2).md` ...
  - 最大試行回数: 99

### コピー完了後の動作
- タブのパスをコピー先パスに更新
- `isExternal = false` に変更
- ファイルツリーに新ファイルを追加

## BR-U3-07: セッション復元

### 復元対象
- ワークスペースパス
- 開いていたタブ（パス、カーソル位置、スクロール位置）
- アクティブだったタブ
- サイドバーの展開フォルダは復元しない（Q9回答C）

### 復元時のエラーハンドリング
- ワークスペースが存在しない: ウェルカム画面表示、最近のワークスペース一覧から選択可能
- タブのファイルが存在しない: そのタブはスキップ（通知表示なし）
- 部分的な復元: 復元できたタブのみ表示

### 保存タイミング
- タブ開閉時
- タブ切替時
- ファイル保存時
- アプリ終了時

## BR-U3-08: ファイル種別判定

### Markdownファイル
- 拡張子: `.md`, `.mdx`
- エディターモード: WYSIWYG（Tiptap経由）

### プレーンテキストファイル
- 拡張子: `.md`, `.mdx` 以外のテキストファイル
- 判定: UTF-8として読み込み可能で、NULLバイト（0x00）を含まない
- エディターモード: プレーンテキスト（monospaceフォント、行番号なし）
- 制限: WYSIWYG機能（ツールバー、スラッシュコマンド）は無効化

### バイナリファイル
- 判定: NULLバイトを含む、または読み込みエラー
- 動作: 開けない（通知「バイナリファイルは開けません」）

---

## BR-U3-09: ドラッグ&ドロップタブ並び替え

### 操作
- タブバー内でのD&D: タブの並び順変更
- ドラッグ中の視覚フィードバック: ドロップ位置にインジケーター表示

### 制約
- ファイルツリーへのD&D（ファイル移動）はMVP対象外
- 外部からのファイルD&DはMVP対象外（ワークスペース外ファイルインポートはファイルオープンダイアログ経由）

---

## BR-U3-10: 新規ファイル作成

### 作成方法
1. **「+」ボタン**: サイドバー上部のボタン → ファイル名入力ダイアログ
2. **コンテキストメニュー**: 右クリック → 「新規ファイル」/「新規フォルダ」→ インライン編集

### インライン編集
- ファイルツリー内にテキスト入力フィールドが出現
- Enter で確定、Escape でキャンセル
- フォーカスアウトで確定

### 「+」ボタンのダイアログ
- ファイル名入力
- 種別選択: ファイル or フォルダ
- 作成先: 現在選択中のフォルダ（なければワークスペースルート）

### デフォルトファイル名
- 新規ファイル: `untitled.md`（重複時は `untitled(1).md`）
- 新規フォルダ: `new-folder`（重複時は `new-folder(1)`）
