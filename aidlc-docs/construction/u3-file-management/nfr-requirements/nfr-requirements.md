# U3: File Management — NFR Requirements

## パフォーマンス要件

### NFR-U3-01: ファイルツリー読み込み速度
- **初回読み込み**: ワークスペースルート直下のエントリ取得 ≤ 100ms（100ファイル/フォルダ以下）
- **遅延読み込み**: サブフォルダ展開時の子ノード取得 ≤ 100ms（100エントリ以下）
- **フィルタリング**: フィルタ適用/解除 ≤ 16ms（1フレーム以内、クライアントサイド処理）
- **ソート**: ディレクトリ優先 + アルファベット順ソート ≤ 10ms（100エントリ以下）
- **規模想定**: 小規模（〜100ファイル）をメインターゲット

### NFR-U3-02: ファイル操作レスポンス
- **ファイル読み込み**: ≤ 100ms（1MB以下、U1 NFR-U1-03準拠）
- **ファイル保存**: ≤ 100ms（200KB以下）、≤ 300ms（200KB〜1MB、性能劣化許容）
- **リネーム**: ≤ 50ms
- **削除**: ≤ 50ms（削除自体。確認ダイアログのUI表示は別）
- **新規ファイル作成**: ≤ 50ms（空ファイル作成）
- **新規フォルダ作成**: ≤ 50ms
- **コピー**: ≤ 200ms（200KB以下）、≤ 500ms（200KB〜1MB、性能劣化許容）
- **移動**: ≤ 100ms（同ドライブ内。クロスドライブ移動はコピー+削除となるため≤ 500ms）
- **ワークスペースクローズ**: ≤ 500ms（セッション保存+全タブクローズ+ツリークリア）
- **クリップボード操作**（copyPath/copyRelativePath）: ≤ 50ms

### NFR-U3-03: タブ切替速度
- **タブ切替**: ≤ 100ms（現タブ状態保存 + 新タブ内容設定 + カーソル/スクロール復元）
- **タブ開く**: ≤ 200ms（ファイル読み込み + EditorCore設定）
- **タブ閉じ**: ≤ 50ms（未保存確認ダイアログのUI表示は別）
- **D&D並び替え**: 60fps（ドラッグ中のアニメーション）

### NFR-U3-04: セッション復元速度
- **セッション読み込み**: ≤ 50ms（SettingsManagerからのJSON読み込み+パース）
- **タブ復元（アクティブタブ）**: ≤ 200ms（ファイル読み込み + EditorCore設定）
- **タブ復元（非アクティブタブ）**: 遅延読み込み（初回切替時に読み込み、起動時には処理しない）
- **起動時の初期表示**: ≤ 500ms（セッション読み込み + ワークスペースオープン + アクティブタブ1つ復元）
- **起動バジェット**: U1 NFR-U1-01の全体1秒枠内。U1=460ms + U2=200ms で660ms使用、U3は残り340msのうち最大200msを使用（U4に140ms確保）

### NFR-U3-05: 仮想スクロール
- **対象**: 1フォルダ内に200+エントリがある場合に仮想スクロール発動
- **DOM数**: 画面表示分 + バッファ（合計最大100ノード）
- **スクロール時のFPS**: ≥ 60fps
- **メモリ使用量（ストレステスト）**: 10,000ノードのツリーでも ≤ 50MB
- **注**: メインターゲットは〜100ファイルの小規模ワークスペース。200+エントリのフォルダは例外ケースとして仮想スクロールで対応。10,000ノードは境界テスト用の上限値。
- **実装制約**: 固定高さアイテムを前提（CSSで各ノードの高さを統一すること）

### NFR-U3-06: ファイル監視
- **変更検出遅延**: ≤ 2,000ms（外部変更から検出までの遅延）
- **バッチ処理**: 短時間の連続変更（例: git操作）はデバウンス（500ms）してまとめて処理
- **CPU負荷**: アイドル時のファイル監視によるCPU使用率 ≤ 1%

## 信頼性要件

### NFR-U3-07: ファイルシステムエラーハンドリング
- **エラー種別**: NOT_FOUND, PERMISSION_DENIED, ALREADY_EXISTS, IO_ERROR, DISK_FULL
- **リトライ**: ファイルI/Oエラーはリトライしない（U1 NFR-U1-05準拠、即座にエラー返却）
- **通知**: 全エラーでToast通知表示（エラーコード + 人間可読メッセージ）
- **アプリ安定性**: ファイルI/Oエラーでアプリがクラッシュしないこと
- **ロックファイル**: ファイルが他プロセスにロックされている場合、適切なエラーメッセージ表示

### NFR-U3-08: 未保存変更の保護（クラッシュリカバリ）
- **リカバリファイル**: 未保存の変更がある場合、30秒ごとにリカバリファイルをtmpディレクトリに書き出し
- **リカバリファイル形式**: `{workspace}/.markdown-editor-recovery/{filename}.recovery`
- **起動時復旧**: アプリ起動時にリカバリファイルの存在をチェック → 見つかった場合は復旧ダイアログ表示
- **クリーンアップ**: 正常保存後/正常終了後にリカバリファイル削除
- **リカバリファイルサイズ**: 元ファイルと同サイズ（圧縮なし）
- **U2自動保存との関係**:
  - **autoモード**: U2 EditorCoreの自動保存がファイルに直接書き込むため、リカバリファイルは生成しない（isDirtyが常にfalseのため）
  - **manualモード**: U2の自動保存はメモリ上のみ → U3のリカバリファイルが未保存コンテンツのクラッシュ保護を担う
  - 両メカニズムは排他的に動作し、競合しない

### NFR-U3-09: セッション永続化の信頼性
- **永続化方式**: SettingsManager経由でJSONファイルに保存
- **破損復旧**: JSONパースエラー時はデフォルト状態で起動（リカバリ不要）
- **アトミック書き込み**: 一時ファイルに書き込み → リネーム（書き込み中のクラッシュ対策）
- **バックアップ**: セッションファイルの直前バージョンを`.bak`として保持

### NFR-U3-10: 外部変更検出と競合解決
- **検出方式**: Tauriのファイル監視API（`@tauri-apps/plugin-fs` watch機能）
- **変更検出時の動作**:
  - タブが未編集の場合: 自動的に最新内容に更新（通知表示）
  - タブが編集中（isDirty）の場合: 確認ダイアログ表示
    - 「外部の変更を読み込む」→ 外部内容で上書き（ローカル変更は破棄）
    - 「ローカルの変更を保持」→ 外部変更を無視（次回保存時に上書き）
    - 「マージ」→ MVP対象外（将来拡張）
- **削除検出**: 開いているファイルが外部で削除された場合、タブにWarningアイコン + 通知
- **フォルダ変更検出**: ワークスペース内のフォルダ構造変更（作成/削除/リネーム）を検出してファイルツリーを自動更新

## セキュリティ要件

### NFR-U3-11: パストラバーサル防止（多層防御）
- **Layer 1 — FileManagerレベル**:
  - 全パス操作でワークスペースルートからの相対パスを正規化
  - `..` を含むパス、ワークスペースルート外のパスを拒否
  - シンボリックリンクのフォロー禁止（ワークスペース外への参照防止）
- **Layer 2 — FileSystemAdapterレベル**:
  - 全I/O操作でパスがワークスペースルート配下であることをダブルチェック
  - Windowsの代替データストリーム（`file.txt:stream`）をブロック
- **Layer 3 — Tauriスコープ**:
  - `tauri.conf.json`のallowlistでワークスペースフォルダのみアクセス許可
  - 動的スコープ更新（ワークスペース変更時にスコープを更新）

### NFR-U3-12: ファイル名サニタイズ
- 新規作成/リネーム時に禁止文字を検出してエラー返却
- HTMLタグ/スクリプトインジェクションの防止（ファイル名をUI表示する際のエスケープ）
- NULLバイトインジェクション防止

### NFR-U3-13: 外部ファイルインポートの安全性
- インポート前にファイルサイズ確認（上限: 1MB、U2準拠）
- バイナリファイル検出（NULLバイトチェック）
- シンボリックリンクの検出と拒否

## ユーザビリティ要件

### NFR-U3-14: WCAG 2.1 AAA準拠（ファイルツリー）
- **ロール/ARIA**:
  - ツリーコンテナ: `role="tree"`, `aria-label="ファイルエクスプローラー"`
  - ノード: `role="treeitem"`, `aria-expanded`（フォルダ）, `aria-selected`, `aria-level`
  - グループ: `role="group"`（子ノードコンテナ）
- **キーボードナビゲーション**:
  - `ArrowUp/Down`: ノード間移動
  - `ArrowRight`: フォルダ展開 / 子ノードに移動
  - `ArrowLeft`: フォルダ折り畳み / 親ノードに移動
  - `Enter`: ファイル開く / フォルダ展開トグル
  - `Home/End`: 先頭/末尾ノード
  - `*`: 同レベルの全フォルダ展開
  - Type-ahead: 文字入力でマッチするノードにフォーカス
- **コントラスト**: 選択状態、ホバー状態、フォーカスリングのコントラスト比 7:1以上
- **フォーカス管理**: フォルダ展開/折り畳み後もフォーカスを維持

### NFR-U3-15: WCAG 2.1 AAA準拠（タブバー）
- **ロール/ARIA**:
  - コンテナ: `role="tablist"`, `aria-label="開いているファイル"`
  - タブ: `role="tab"`, `aria-selected`, `aria-controls`
  - パネル: `role="tabpanel"`, `aria-labelledby`
- **未保存状態の通知**: `aria-label`に未保存状態を含む（例: "article.md（未保存の変更があります）"）
- **キーボードナビゲーション**:
  - `ArrowLeft/Right`: タブ間移動
  - `Home/End`: 先頭/末尾タブ
  - `Delete` or `Ctrl+W`: タブ閉じ
  - `Ctrl+Shift+Left/Right`: タブ並び替え（D&Dのキーボード代替）

### NFR-U3-16: WCAG 2.1 AAA準拠（ダイアログ/コンテキストメニュー）
- **ダイアログ**: `role="dialog"`, `aria-modal="true"`, フォーカストラップ, Escapeで閉じ
- **コンテキストメニュー**: `role="menu"`, `role="menuitem"`, ArrowUp/Downで移動, Escapeで閉じ
- **ライブリージョン**: ファイル操作結果の通知をスクリーンリーダーに伝達（`aria-live="polite"`）

## 保守性要件

### NFR-U3-17: コード品質
- **TypeScript**: strict mode有効（U1/U2と統一）
- **テストカバレッジ**: ≥ 80%（ビジネスロジック + ユーティリティ）
- **テスト種別**:
  - ユニットテスト: FileManager, WorkspaceService, バリデーション, ストア
  - コンポーネントテスト: FileTree, TabBar, ダイアログ
- **リント/フォーマット**: ESLint + Prettier（既存設定準拠）

### NFR-U3-18: FileSystemAdapter抽象化維持
- 全ファイルI/O操作はFileSystemAdapter経由（直接fs APIを呼ばない）
- ファイル監視もFileSystemAdapterにwatch/unwatch APIを追加
- テスト時はFileSystemAdapterをモックに差し替え可能
