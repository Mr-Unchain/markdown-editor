# ユーザーストーリー

## 凡例
- **優先度**: MVP = 初期リリースに含む / Post-MVP = 将来リリース
- **ペルソナ**: T = 田中（テックブロガー） / S = 佐藤（一般ブロガー） / Both = 両方
- **受け入れ基準**: チェックリスト形式

---

## Epic 1: コアエディター体験（WYSIWYG編集）

### US-01: リッチテキスト編集
**As a** ブロガー
**I want to** WYSIWYGモードでMarkdownを意識せずに記事を執筆できる
**So that** 記法を覚えなくても直感的にリッチなコンテンツを作成できる

**優先度**: MVP | **ペルソナ**: Both

**受け入れ基準**:
- [ ] 見出し（H1-H6）をスラッシュコマンドまたはツールバーから挿入できる
- [ ] テキスト選択後、太字・斜体・取り消し線・コードをツールバーまたはショートカットで適用できる
- [ ] 書式がリアルタイムでレンダリングされる（入力即反映）
- [ ] Enter/Backspaceでブロック要素の作成・解除が自然に動作する
- [ ] Markdown記法を直接入力しても自動的にリッチテキストに変換される（例: `**太字**` → **太字**）

### US-02: リスト・チェックリスト編集
**As a** ブロガー
**I want to** 箇条書き、番号付きリスト、チェックリストを視覚的に編集できる
**So that** 構造化されたコンテンツを簡単に作成できる

**優先度**: MVP | **ペルソナ**: Both

**受け入れ基準**:
- [ ] 箇条書きリストを作成・ネスト・解除できる
- [ ] 番号付きリストを作成でき、番号が自動採番される
- [ ] チェックリスト（タスクリスト）を作成し、チェックのON/OFFができる
- [ ] Tab/Shift+Tabでインデント・アンインデントができる
- [ ] リスト内でEnterで新項目追加、空行Enterでリスト解除

### US-03: テーブル編集
**As a** ブロガー
**I want to** テーブルを視覚的に作成・編集できる
**So that** Markdownのテーブル記法を手打ちせずにデータを整理できる

**優先度**: MVP | **ペルソナ**: Both

**受け入れ基準**:
- [ ] 行数×列数を指定してテーブルを挿入できる
- [ ] セルをクリックして直接編集できる
- [ ] 行・列の追加・削除ができる
- [ ] セル内容の左揃え・中央揃え・右揃えを設定できる
- [ ] Tab/Shift+Tabでセル間を移動できる

### US-04: コードブロック編集
**As a** テックブロガー
**I want to** シンタックスハイライト付きのコードブロックを挿入・編集できる
**So that** 技術記事内のコードが読みやすく表示される

**優先度**: MVP | **ペルソナ**: T (主), Both (利用可)

**受け入れ基準**:
- [ ] 言語を指定してコードブロックを挿入できる
- [ ] 主要言語（JavaScript, TypeScript, Python, Rust, Go, HTML, CSS, JSON, YAML, Shell）のシンタックスハイライトが動作する
- [ ] コードブロック内ではエディターのショートカットが干渉しない
- [ ] コードブロックからの脱出が容易にできる（下矢印キーまたはEscape）
- [ ] インラインコード（`code`）もショートカットまたはバッククォートで挿入できる

### US-05: 引用・区切り線・リンク
**As a** ブロガー
**I want to** 引用ブロック、区切り線、リンクを簡単に挿入できる
**So that** 記事のレイアウトを豊かにできる

**優先度**: MVP | **ペルソナ**: Both

**受け入れ基準**:
- [ ] 引用ブロック（blockquote）を挿入し、ネストできる
- [ ] 区切り線（hr）をスラッシュコマンドまたはMarkdown記法（---）で挿入できる
- [ ] テキスト選択後にリンクURLを入力してハイパーリンクを作成できる
- [ ] URLペーストでリンクが自動作成される
- [ ] リンクのURL・テキストを後から編集できる

### US-06: スラッシュコマンド
**As a** ブロガー
**I want to** `/` を入力するとブロック要素の挿入メニューが表示される
**So that** キーボードから手を離さずに各種要素を素早く挿入できる

**優先度**: MVP | **ペルソナ**: Both

**受け入れ基準**:
- [ ] `/` 入力でコマンドパレットが表示される
- [ ] テキスト入力でコマンドをフィルタリングできる
- [ ] 見出し、リスト、コードブロック、テーブル、引用、区切り線、画像の挿入が可能
- [ ] 矢印キー+Enterまたはクリックでコマンドを実行できる
- [ ] Escapeでコマンドパレットをキャンセルできる

### US-07: キーボードショートカット
**As a** ブロガー
**I want to** 主要な操作にキーボードショートカットが設定されている
**So that** 効率的に記事を編集できる

**優先度**: MVP | **ペルソナ**: Both

**受け入れ基準**:
- [ ] Ctrl/Cmd+B で太字、Ctrl/Cmd+I で斜体
- [ ] Ctrl/Cmd+S で保存
- [ ] Ctrl/Cmd+Z でUndo、Ctrl/Cmd+Shift+Z でRedo
- [ ] ショートカット一覧をヘルプから確認できる

### US-08: Undo/Redo
**As a** ブロガー
**I want to** 操作を取り消し・やり直しできる
**So that** 間違った編集をすぐに修正できる

**優先度**: MVP | **ペルソナ**: Both

**受け入れ基準**:
- [ ] Ctrl/Cmd+Z で直前の操作を取り消せる
- [ ] Ctrl/Cmd+Shift+Z でやり直しができる
- [ ] 複数ステップのUndo/Redoが動作する
- [ ] ファイルを保存してもUndo履歴が保持される（セッション中）

## Epic 2: ファイル・プロジェクト管理

### US-09: ワークスペース管理
**As a** ブロガー
**I want to** フォルダをワークスペースとして開き、記事を一覧管理できる
**So that** すべての記事を一つの場所で整理できる

**優先度**: MVP | **ペルソナ**: Both

**受け入れ基準**:
- [ ] フォルダを選択してワークスペースとして開ける
- [ ] サイドバーにフォルダツリーが表示される
- [ ] ファイル・フォルダの新規作成ができる
- [ ] ファイルをクリックしてエディターで開ける
- [ ] 最後に開いたワークスペースを起動時に復元できる

### US-10: ファイル操作
**As a** ブロガー
**I want to** ファイルの作成、保存、名前変更、削除ができる
**So that** 記事ファイルを自由に管理できる

**優先度**: MVP | **ペルソナ**: Both

**受け入れ基準**:
- [ ] 新規Markdownファイルを作成できる
- [ ] Ctrl/Cmd+S で現在のファイルを保存できる
- [ ] 未保存の変更がある場合、タブにインジケーターが表示される
- [ ] ファイル名の変更ができる
- [ ] ファイルの削除ができる（確認ダイアログ付き）
- [ ] ウィンドウを閉じる際、未保存の変更について警告される

### US-11: 複数タブ編集
**As a** ブロガー
**I want to** 複数のファイルをタブで同時に開いて切り替えられる
**So that** 関連する記事を参照しながら執筆できる

**優先度**: MVP | **ペルソナ**: Both

**受け入れ基準**:
- [ ] 複数のファイルをタブで開ける
- [ ] タブをクリックしてファイルを切り替えられる
- [ ] タブを閉じることができる（未保存の場合は確認）
- [ ] 開いているタブの一覧が見える

## Epic 3: プラットフォーム連携（投稿・エクスポート）

### US-12: Zenn記事投稿
**As a** テックブロガー
**I want to** エディターからZennに直接記事を下書き投稿できる
**So that** コピー＆ペーストの手間なく記事を公開できる

**優先度**: MVP | **ペルソナ**: T (主)

**受け入れ基準**:
- [ ] Zennアカウントとの認証・接続ができる
- [ ] 記事のタイトル、トピック、公開設定を指定できる
- [ ] エディターから直接Zennに下書き保存できる
- [ ] 投稿済み記事の更新（再投稿）ができる
- [ ] 投稿成功/失敗の通知が表示される

### US-13: note記事投稿
**As a** 一般ブロガー
**I want to** エディターからnoteに記事を投稿できる
**So that** 別のエディターからnoteに移動する手間が省ける

**優先度**: Post-MVP | **ペルソナ**: S (主)

**受け入れ基準**:
- [ ] noteアカウントとの認証・接続ができる
- [ ] 記事を下書きとして投稿できる
- [ ] 画像がnoteの画像ストレージに自動アップロードされる
- [ ] 投稿成功/失敗の通知が表示される

### US-14: microCMSコンテンツ投稿
**As a** ブロガー
**I want to** エディターからmicroCMSに記事コンテンツを投稿できる
**So that** ヘッドレスCMSへの記事登録を効率化できる

**優先度**: Post-MVP | **ペルソナ**: Both

**受け入れ基準**:
- [ ] microCMSのサービスID・APIキーを設定できる
- [ ] 記事をmicroCMSのコンテンツとして投稿（下書き/公開）できる
- [ ] 画像がmicroCMSの画像ストレージに自動アップロードされる
- [ ] コンテンツIDの紐付けにより既存記事の更新ができる

### US-15: Contentfulコンテンツ投稿
**As a** ブロガー
**I want to** エディターからContentfulにコンテンツを投稿できる
**So that** 国際的なヘッドレスCMSでもコンテンツを管理できる

**優先度**: Post-MVP | **ペルソナ**: Both

**受け入れ基準**:
- [ ] ContentfulのSpace ID・アクセストークンを設定できる
- [ ] 記事をContentfulのエントリとして投稿できる
- [ ] 画像がContentfulのAssetとして自動アップロードされる

### US-16: プラットフォーム向けエクスポート
**As a** ブロガー
**I want to** API非対応のプラットフォーム向けにフォーマット変換してコピーできる
**So that** どのプラットフォームにもコンテンツを投稿できる

**優先度**: MVP | **ペルソナ**: Both

**受け入れ基準**:
- [ ] HTMLとしてコピーできる（リッチエディタへのペースト用）
- [ ] プレーンMarkdownとしてコピーできる
- [ ] HTMLファイルとしてダウンロードできる
- [ ] コピー成功の通知が表示される

## Epic 4: 画像管理

### US-17: 画像挿入
**As a** ブロガー
**I want to** ドラッグ&ドロップまたはペーストで画像を記事に挿入できる
**So that** 画像を簡単にコンテンツに含められる

**優先度**: MVP | **ペルソナ**: Both

**受け入れ基準**:
- [ ] ドラッグ&ドロップで画像ファイルを挿入できる
- [ ] クリップボードからの画像ペーストができる
- [ ] スラッシュコマンドから画像ファイルを選択して挿入できる
- [ ] 挿入した画像がエディター内でプレビュー表示される
- [ ] 画像ファイルはワークスペース内のフォルダに保存される

### US-18: プラットフォーム画像アップロード
**As a** ブロガー
**I want to** プラットフォームに投稿する際に画像が自動アップロードされる
**So that** 画像を手動でアップロードし直す必要がない

**優先度**: MVP（Zenn連携時） | **ペルソナ**: Both

**受け入れ基準**:
- [ ] Zenn投稿時にローカル画像がZennの画像ストレージにアップロードされる
- [ ] アップロード後、記事内の画像パスが自動的にアップロード先URLに置換される
- [ ] アップロードの進捗が表示される
- [ ] アップロード失敗時にエラーメッセージが表示される

---

## Epic 5: プラグインシステム

### US-19: プラグインの管理
**As a** テックブロガー
**I want to** プラグインをインストール・有効化・無効化できる
**So that** 必要な機能だけを追加してエディターを軽量に保てる

**優先度**: Post-MVP | **ペルソナ**: T (主)

**受け入れ基準**:
- [ ] 設定画面からプラグイン一覧を確認できる
- [ ] プラグインの有効/無効を切り替えられる
- [ ] プラグインの有効/無効変更後、即座に反映される
- [ ] 無効化されたプラグインは起動時にロードされない

### US-20: Markdown構文プラグイン
**As a** テックブロガー
**I want to** 数式やMermaidダイアグラムなどの拡張構文をプラグインとして追加できる
**So that** 技術記事に必要な表現を自由に使える

**優先度**: Post-MVP | **ペルソナ**: T

**受け入れ基準**:
- [ ] 数式プラグイン（KaTeX）を有効にするとインライン・ブロック数式が使用可能
- [ ] Mermaidプラグインを有効にするとダイアグラムがプレビューされる
- [ ] プラグインが遅延読み込みされ、起動速度に影響しない
- [ ] プラグインが無効な状態ではrawテキストとして表示される

### US-21: プラットフォーム構文プラグイン
**As a** テックブロガー
**I want to** Zenn独自の記法（メッセージ、アコーディオン等）をプラグインで対応できる
**So that** プラットフォーム固有のリッチコンテンツを作成できる

**優先度**: Post-MVP | **ペルソナ**: T

**受け入れ基準**:
- [ ] Zennのメッセージ記法（:::message）がWYSIWYGで視覚的に編集できる
- [ ] Zennのアコーディオン記法（:::details）が視覚的に編集できる
- [ ] プラグインが無効な状態ではrawテキストとして表示される

## Epic 6: アプリケーション設定・カスタマイズ

### US-22: プラットフォーム接続設定
**As a** ブロガー
**I want to** 各プラットフォームの認証情報を設定画面から管理できる
**So that** プラットフォーム連携を安全にセットアップできる

**優先度**: MVP | **ペルソナ**: Both

**受け入れ基準**:
- [ ] 設定画面からプラットフォーム一覧を確認できる
- [ ] 各プラットフォームのAPIキー・認証情報を入力・保存できる
- [ ] 認証情報はセキュアに保存される（平文で保存されない）
- [ ] 接続テスト機能で認証が有効か確認できる

### US-23: エディター表示設定
**As a** ブロガー
**I want to** フォントサイズ、テーマなどの表示設定を変更できる
**So that** 自分に合った環境で執筆できる

**優先度**: Post-MVP | **ペルソナ**: Both

**受け入れ基準**:
- [ ] フォントサイズを変更できる
- [ ] ライト/ダークテーマを切り替えられる
- [ ] エディターの幅（文字数）を設定できる
- [ ] 設定がアプリ再起動後も保持される
